What we are after: \(q(\theta\mid x)\)

A simulator gives us pairs \((\theta, x)\): parameter in, observation out. Plotted together they form a joint cloud.

Our job is to describe the cloud in the \(\theta\) direction: at each value of \(x\), what is the distribution of \(\theta\) consistent with it?

\[ q(\theta\mid x) \;=\; \text{vertical slice of the cloud at } x \]

Move the slider: the orange band picks out a slice; the points inside are samples of \(q(\theta\mid x_\mathrm{obs})\).

For now we assume \(q(\theta\mid x)\) is uni-modal at every \(x\). The multi-modal case waits until the last section.

The simplest model for \(q(\theta\mid x)\)

Fit a curve \(\mu_\theta(x)\) through the cloud and assume the slice at every \(x\) is Gaussian with the same width \(\sigma_\theta\).

\[ q_\phi(\theta\mid x) \;=\; \mathcal{N}\!\bigl(\theta\,\big|\,\mu_\theta(x),\,\sigma_\theta^{2}\bigr) \]

Trainable parameters \(\phi = (\mathbf{w}, \sigma_\theta)\): the curve and the band width.

Black line: \(\mu_\theta(x)\). Shaded: \(\mu_\theta(x) \pm 2\sigma_\theta\).

The linear-basis picture

Pick basis functions \(\phi_j(x)\) that encode plausible functional behaviour. Combine \(M\) of them with free parameters:

\[ \mu_\theta(x; \mathbf{w}) \;=\; \sum_{j=0}^{M-1} w_j\,\phi_j(x) \;=\; \mathbf{w}^T\boldsymbol{\phi}(x), \qquad \mathbf{w} \in \mathbb{R}^M \]
  • Basis functions \(\phi_j(x)\) — fixed templates capturing expected shapes
  • Weight vector \(\mathbf{w} = (w_0, \ldots, w_{M-1})^T\) — \(M\) free parameters, fitted to data
  • "Linear" refers to linearity in \(\mathbf{w}\), not in \(x\) — the templates themselves can be nonlinear

The modelling decision is the choice of \(\boldsymbol{\phi}\). Everything that follows in this lecture is one continuous attack on that choice.

From model likelihood to loss function

With independent Gaussian noise, the joint likelihood factorises:

\[ p(\boldsymbol\theta \mid \mathbf{w}, \sigma_\theta) \;=\; \prod_{n=1}^{N} \mathcal{N}\!\bigl(\theta_n \,\big|\, \mu_\theta(x_n;\mathbf{w}),\, \sigma_\theta^{2}\bigr) \]

Take the log and write the constant explicitly in terms of \(N\) and \(\sigma_\theta\):

\[ \ln p(\boldsymbol\theta \mid \mathbf{w}, \sigma_\theta) \;=\; -\frac{N}{2}\ln\!\bigl(2\pi\sigma_\theta^{2}\bigr) \;-\; \frac{1}{2\sigma_\theta^{2}}\sum_{n=1}^{N}\bigl(\theta_n - \mu_\theta(x_n;\mathbf{w})\bigr)^{2} \]

Negate to turn maximising the likelihood into minimising a loss:

\[ E(\mathbf{w}, \sigma_\theta) \;=\; \frac{N}{2}\ln\!\bigl(2\pi\sigma_\theta^{2}\bigr) \;+\; \frac{1}{2\sigma_\theta^{2}}\sum_{n=1}^{N}\bigl(\theta_n - \mu_\theta(x_n;\mathbf{w})\bigr)^{2} \]

Solving for the weights and the variance

Split the loss into a \(\sigma_\theta\)-independent shape \(S(\mathbf{w})\) and a \(\sigma_\theta\)-only piece:

\[ E(\mathbf{w}, \sigma_\theta) \;=\; \frac{N}{2}\ln\!\bigl(2\pi\sigma_\theta^{2}\bigr) \;+\; \frac{S(\mathbf{w})}{2\sigma_\theta^{2}}, \qquad S(\mathbf{w}) = \sum_{n}\bigl(\theta_n - \mathbf{w}^T\boldsymbol{\phi}(x_n)\bigr)^{2} \]

Step 1 — weights. The \(\sigma_\theta\) prefactor is positive, so minimising \(E\) in \(\mathbf{w}\) is the same as minimising \(S(\mathbf{w})\). Set \(\nabla_\mathbf{w} S = 0\); with the design matrix \(\boldsymbol\Phi_{nj} = \phi_j(x_n)\),

\[ \mathbf{w}_{\mathrm{ML}} \;=\; (\boldsymbol\Phi^T\boldsymbol\Phi)^{-1}\boldsymbol\Phi^T\boldsymbol\theta \]

Step 2 — variance. Plug \(\mathbf{w}_{\mathrm{ML}}\) back and set \(\partial E/\partial\sigma_\theta = 0\):

\[ \frac{N}{\sigma_\theta} \;-\; \frac{S(\mathbf{w}_{\mathrm{ML}})}{\sigma_\theta^{3}} \;=\; 0 \quad\Longrightarrow\quad \sigma_\theta^{2} \;=\; \frac{1}{N}\sum_{n}\bigl(\theta_n - \mu_\theta(x_n;\mathbf{w}_{\mathrm{ML}})\bigr)^{2} \]

The MLE divides by \(N\). The unbiased estimator divides by \(N-M\) (Bessel-style correction for the \(M\) fitted weights); the two agree as \(N \to \infty\).

In \(\mu(x) = \mathbf{w}^T\boldsymbol{\phi}(x)\), linear MLE solves the easy part — the weights. All of the modelling burden is on \(\boldsymbol{\phi}\).

Can we learn the basis functions themselves?

Neural networks + SGD

Replace fixed templates with parametric ones. Train them by gradient descent.

Towards learnable building blocks

Replace each fixed template by a parametric one:

\[ \phi_j(x) \;=\; g\!\bigl(\mathbf{v}_j^T x + b_j\bigr) \]

\(g\) = nonlinear activation; \(\mathbf{v}_j, b_j\) are learnable.

Grey: individual \(\phi_j(x)\). Each one moves with \(\mathbf{v}_j, b_j\) — the basis is now a free parameter.

One hidden layer = learnable \(\boldsymbol{\phi}\)

Stack \(H\) neurons and let a linear head combine them:

\[ \mu(x;\mathbf{w},\mathbf{v},\mathbf{b}) \;=\; \sum_{j=1}^{H} w_j\,\underbrace{g(\mathbf{v}_j^T x + b_j)}_{\text{learnable }\phi_j(x)} + w_0 \]

This is exactly the linear-basis model \(\mu_\theta(x) = \mathbf{w}^T\boldsymbol{\phi}(x)\) — only now \(\boldsymbol{\phi}\) has its own parameters that we will train jointly with \(\mathbf{w}\).

Universal approximation — one line

With enough neurons, a single hidden layer can approximate any continuous function on a compact domain to arbitrary precision.

Cybenko 1989, Hornik 1991.

  • Why we care. Whatever \(\mu(x)\) we secretly need, the network can in principle express it. The basis is no longer the bottleneck.
  • Why we still need engineering. The theorem says weights exist; it does not say SGD will find them. Depth and inductive bias matter in practice.

No closed form for the weights any more — we need gradient descent.

Enter gradient descent

\[ \mathbf{w} \;\leftarrow\; \mathbf{w} - \eta\,\nabla E(\mathbf{w}) \]

\(\eta\) = learning rate. Step downhill.

Toy loss \(E(w) = \tfrac{1}{2}w^2 + 0.3\sin(3w)\). Small \(\eta\): crawls. Right \(\eta\): converges fast. Large \(\eta\): bounces or diverges.

GD — details & pseudocode

  • \(\nabla E(\mathbf{w})\) points in the direction of steepest increase. We step opposite.
  • Optimal \(\eta\) scales like \(1/E''(\mathbf{w})\) (inverse curvature). Anisotropic curvature is the practical headache.
  • Full-batch GD uses all \(N\) samples per step — expensive for large datasets.
// Given training data {(x_n, t_n)}, loss E, learning rate η

initialise w randomly

for step = 1, 2, ... until converged:
    // Compute gradient over all N samples
    g ← (1/N) Σ_n ∇_w E_n(w)

    // Update
    w ← w − η · g

Mini-batch SGD

Computing the gradient over all \(N\) samples per step is wasteful. Use a random mini-batch \(\mathcal{B}\) of size \(B\ll N\):

\[ \nabla E(\mathbf{w}) \;\approx\; \frac{1}{B}\sum_{n\in\mathcal{B}} \nabla E_n(\mathbf{w}) \]
  • Large \(B\): precise gradient, expensive per step.
  • Small \(B\): noisy gradient, cheap per step; the noise often helps generalisation.
  • One full sweep through the data = one epoch. Typical \(B \in [32, 256]\).
  • "SGD" almost always means "mini-batch SGD".

Backprop — one slide

Backpropagation is just the chain rule, applied bottom-up on a computation graph. For each parameter \(w_\ell\) we want \(\partial E / \partial w_\ell\); we get it by sweeping local Jacobians backwards from the loss.

You do not have to write it. Modern libraries (PyTorch, JAX) build the graph as you call functions and differentiate it for you — this is autograd.

Practically: you write the forward pass, call loss.backward(), get gradients of every parameter. Cost ≈ 2× the forward pass.

Practical optimisers in one slide

Adam — the default optimiser

  • Keeps running averages of the gradient (\(m_t\)) and its square (\(v_t\)).
  • Effective per-parameter step \(\propto m_t/\sqrt{v_t}\): rescales each direction by its own curvature.
  • Robust default: \(\eta = 10^{-3}\), \(\beta_1=0.9\), \(\beta_2=0.999\).

Learning-rate schedule

  • Decay \(\eta\) as training progresses (cosine, step, or linear warmup-then-decay).
  • Helps SGD settle into the minimum once you are close.
  • Cheap, large effect. Always worth tuning.

Plenty more (RMSProp, AdamW, Lion, SOAP, …). Adam + a sensible schedule is a strong default for everything in this school.

From Gaussian to flexible \(q_\phi(\theta\mid x)\)

Let the noise depend on \(x\); then drop the Gaussian assumption altogether.

One slide on classification: Bayes ↔ sigmoid

Train a binary classifier on \(y\in\{0,1\}\) with cross-entropy. With enough data and capacity, the optimal output is:

\[ p(y\!=\!1\mid x) \;=\; \sigma\!\bigl(f_\phi(x)\bigr), \qquad f_\phi(x)^\star = \ln\!\frac{p(x\mid y\!=\!1)}{p(x\mid y\!=\!0)} \]

The logit is a log-density ratio. Classifiers are statistics in disguise.

Plug \(p(y\!=\!1) = p(y\!=\!0)\) and you get \(p(y\!=\!1\mid x) - 1/2 = \tfrac{1}{4}f^\star(x) + \mathcal{O}(f^{\star 2})\). The same network can target probabilities, densities, or ratios.

Why this matters for SBI: this is the doorway to neural ratio estimation (NRE), the classifier-flavoured cousin of NPE. Lecture 3 names it; we mention it here once and move on.

Heteroscedastic noise — \(\sigma^2(x)\) as a second output

Our first model baked in a single \(\sigma_\theta\). Make \(\sigma\) a function of \(x\); same network, two heads: \(\mu_\phi(x)\) and \(\log\sigma_\phi(x)\). The NLL trades off mean accuracy against calibrated width.

\[ q_\phi(\theta\mid x) \;=\; \mathcal{N}\!\bigl(\theta \,\big|\, \mu_\phi(x), \sigma_\phi(x)^2\bigr), \qquad \mathcal{L} = -\!\sum_n \log q_\phi(\theta_n\mid x_n) \]

Toy: noise grows toward the edges. Constant-σ fit (blue band) overconfident in the middle, underconfident at edges; learned \(\sigma(x)\) (orange) tracks the true scale.

The Gaussian-head \(q_\phi(\theta\mid x)\) — the destination of this lecture

Density family.

\( q_\phi(\theta\mid x) \;=\; \mathcal{N}\!\bigl(\theta\,\big|\,\boldsymbol\mu_\phi(x), \boldsymbol\Sigma_\phi(x)\bigr) \)

Both mean and covariance are NN outputs — the network learns where to be confident.

Loss on \((\theta_n, x_n)\) pairs:

\( \mathcal{L}(\phi) = -\tfrac{1}{N}\sum_n \log q_\phi(\theta_n\mid x_n) \)

For diagonal \(\Sigma\): \(\;\tfrac{1}{2}\bigl((\theta-\mu)/\sigma\bigr)^2 + \log\sigma\). Constant \(\Sigma\) reduces to MSE — the linear-regression case.

// Supervised training of q_φ(θ|x)
// Given pairs (θ_n, x_n)
for step = 1, 2, ... :
    sample mini-batch B of pairs
    μ, log_σ = f_φ(x)
    L = mean( ½((θ−μ)/σ)² + log_σ )
    φ ← Adam(φ, ∇_φ L)
Where Lec 3 picks this up. If \((\theta_n, x_n)\) come from a simulator (\(\theta_n\sim p(\theta), x_n\sim p(x\mid\theta_n)\)), the same procedure is neural posterior estimation (NPE).

Live demo — Gaussian-head NN on \(\theta = a + b\,x\)

Forward model: \(t_i = a + b\,x_i + \varepsilon\), 20-point grid, \(\varepsilon\sim\mathcal{N}(0,0.25^2)\). Priors \(a\sim\mathcal U[-1,1]\), \(b\sim\mathcal U[-2,2]\). Network: MLP \([20{\to}32{\to}16{\to}5]\), output \((\mu_a,\mu_b,\log\sigma_a,\log\sigma_b,\rho)\).

Where Gaussian breaks: multimodal posteriors

The Gaussian head was a modelling choice we put in by hand — just like the polynomial basis at the start. It fails when the true posterior is not Gaussian.

True posterior
multimodal, curved, heavy-tailed
Best Gaussian fit
smears across modes, miscalibrated tails
What we want
a flexible density that takes any shape but is still a proper PDF

Wishlist: ① evaluate \(\log q_\phi\); ② sample; ③ arbitrary shape. Normalising flows deliver all three.

Normalising flows — one-slide preview

Idea. Take a simple base density (Gaussian noise \(z\)). Push it through a stack of invertible neural maps to land on a complex target \(\theta\).

\( \theta = f_L\circ\cdots\circ f_1(z), \quad z\sim\mathcal{N}(0,I) \)

Change-of-variables. For invertible \(f\),

\( q_\phi(\theta\mid x) = p_Z(z)\;\bigl|\det \partial f/\partial z\bigr|^{-1} \)

Build \(f\) so the Jacobian determinant is cheap (triangular).

Full machinery — coupling layers, conditional flows, log-Jacobian sums — in Lecture 3.

Base \(z\sim\mathcal{N}(0,1.5^2)\) warped by \(\theta = z + A\sin z\). As \(A\to 1\), the same base produces a bimodal target — for free.
