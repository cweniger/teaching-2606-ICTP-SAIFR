Lecture 4a: CNNs for image analysis in SBI

From raw image \(x\) to a learned summary \(s_\psi(x)\) feeding \(q_\phi(\theta\mid s_\psi(x))\)

Christoph Weniger — University of Amsterdam (GRAPPA)

Motivation — Strong Lensing as Image-Based SBI

The science: a massive foreground galaxy bends light from a background source into arcs and Einstein rings. The image \(x\) constrains the lens mass profile, source position, and potentially the presence of dark matter substructure.

The simulator: given parameters \(\theta\) (lens mass, ellipticity, source profile, substructure mass function, ...), ray-trace photons and add a realistic PSF and noise to produce a pixel image \(x \in \mathbb{R}^{C\times H\times W}\).

The inference task: recover \(p(\theta\mid x)\). With \(x\) running into \(10^4\)–\(10^6\) pixels, a flat MLP on \(\mathrm{vec}(x)\) is hopeless. We need an architecture that respects spatial structure.

Why this matters today:

  • Euclid, LSST, JWST will deliver \(10^5\)+ strong lenses.
  • Each lens is an image. No useful low-dimensional summary statistic exists for substructure.
  • SBI with learned summaries \(s_\psi(x)\) is the natural fit, and \(s_\psi\) is a CNN.

Today: how a CNN turns raw pixels into a feature vector we can hand to \(q_\phi(\theta\mid s_\psi(x))\).

Why Not Just Flatten and Use an MLP?

What we had so far:

  • Low-dimensional input \(\mathbf{x} \in \mathbb{R}^d\)
  • MLP \(\rightarrow\) logits / regression head
  • Cross-entropy or NLL loss

What an image needs:

  • Even a tiny \(3\times 32\times 32\) image is \(3072\) values; a real lens image is \(10^4\)–\(10^6\).
  • A flat MLP has no notion of locality or translation.
  • Number of parameters explodes; nothing is shared across pixels.

We need something before the MLP that extracts features while respecting spatial structure. That something is the convolutional neural network.

What Is a Digital Image?

A colour image is a 3D tensor: 3 channels (Red, Green, Blue) × height × width. Each channel is a grayscale intensity map.

Shape in PyTorch: (batch, 3, H, W) — e.g. a 64×64 RGB image is (1, 3, 64, 64)

For strong-lensing images the "channels" are filter bands (e.g. g, r, i) rather than RGB — the tensor shape is the same.

From Colour Channels to Semantic Channels

  • An RGB (or multi-band) image has a few channels that encode brightness at high spatial resolution.
  • A CNN transforms these into many channels that encode learned semantic concepts (edges, arcs, ring fragments, substructure signatures, ...) at lower resolution. This is the key idea behind CNNs.
  • For SBI on lensing images, these semantic channels are the raw material out of which \(s_\psi(x)\) is built.

2D Convolution — The Basic Idea

A small kernel slides across the input. At each position: element-wise multiply, then sum.

2D Convolution — Multiple Input Channels

The kernel spans all input channels. At each position: multiply per channel, sum everything → one output value.

  • Each kernel has \(C \times h \times w\) parameters and produces one output channel, a feature map. Here \(C{=}h{=}w{=}3\).
  • The kernel sums over all input channels → one scalar per position.
  • ReLU(\(x\)) = max(0, \(x\)) discards negative responses — try it with a gradient kernel to pick one edge direction.
  • MaxPool 2×2 coarse-grains: keeps the strongest response in each 2×2 block, halving resolution.
  • These kernels are hand-crafted examples. In a real CNN, kernels are learned from the data.

2D Convolution — Mathematical Definition

\[\mathbf{o}_{b,d,j,i} = \sum_{c=0}^{C-1} \sum_{n=0}^{h-1} \sum_{m=0}^{w-1} \mathbf{x}_{b,c,\,j+n,\,i+m} \cdot \mathbf{u}_{d,c,n,m}\]

\(\mathbf{x} \in \mathbb{R}^{B \times C \times H \times W}\) (input), \(\mathbf{u} \in \mathbb{R}^{D \times C \times h \times w}\) (kernels), \(\mathbf{o} \in \mathbb{R}^{B \times D \times H' \times W'}\) (output)

\(B\) batch, \(C\) input channels, \(D\) output channels (number of kernels), \(h \times w\) kernel size

\(H' = H - h + 1\), \(W' = W - w + 1\) (without padding)

In PyTorch: nn.Conv2d(in_channels=C, out_channels=D, kernel_size=(h, w))

Weight Sharing — Why a CNN Is Dramatically Cheaper

The same kernel slides across every position. One pattern detector, reused everywhere.

MLP first layer on a 256×256 single-channel lens image, 1024 hidden units:

\(N_\text{MLP} = 256 \times 256 \times 1024 \approx 6.7 \times 10^{7}\)

  • Every pixel gets its own weight into every hidden unit.
  • Nothing is shared. A pattern learned in one corner is invisible elsewhere.
  • Memory and overfitting risk both explode.

Conv layer with 64 kernels of size 3×3 on the same image:

\(N_\text{Conv} = 64 \times (1 \times 3 \times 3) + 64 = 640\)

  • Each kernel is applied at every pixel: weight sharing.
  • Locality: each output uses a tiny 3×3 neighbourhood.
  • ~105× fewer parameters, far better generalisation.

Two priors baked into the architecture: (i) useful patterns are local, and (ii) the rule for detecting them is the same everywhere in the image. Both are obviously true for arcs, edges, and substructure in a lens image.

Why ReLU?

What it does: ReLU(x) = max(0, x) — keeps positive values, sets negative to zero.

Each conv kernel responds positively and negatively. ReLU keeps only the positive activations, so the kernel becomes a feature detector that fires when its pattern is present.

Why we need it: A convolution is a linear operation. Stacking linear layers gives another linear layer, no gain in expressiveness.

ReLU breaks the linearity. With nonlinearity between layers, deeper networks can represent increasingly complex features.

The building block: Conv → ReLU → Pool is the standard unit. Conv extracts, ReLU selects, Pool coarsens.

Max Pooling — Coarse-Graining Spatial Information

MaxPool 2×2 takes the maximum in each 2×2 block. Applied independently per channel. No learned parameters.

 
\(\displaystyle \mathbf{o}_{b,c,j,i} = \max_{n < p,\, m < q} \mathbf{x}_{b,c,\,pj+n,\,qi+m}\)  —  here \(p = q = 2\). In PyTorch: nn.MaxPool2d(2)

Equivariance of Convolution Operations

A function \(f\) is equivariant to \(g\) if \(f(g(\mathbf{x})) = g(f(\mathbf{x}))\).

Weight sharing makes the convolutional layer equivariant to translation: if an object moves in the input image, its representation moves by the same amount in the output. The same arc, anywhere in the field, produces the same feature, just shifted.

Why this matters for lensing: a strong lens can appear anywhere in the image cutout. Equivariance means the CNN does not need to see every translation during training to recognise it at test time.

Credits: LeCun et al, Gradient-based learning applied to document recognition, 1998.

From Pixels to Semantics

CNNs trade spatial resolution for semantic richness: resolution goes down, abstraction goes up.

Input: raw pixel values
\[\mathbf{v}_{i,j}^{(0)} = \alpha_r \,\mathbf{e}_{\text{red}} + \alpha_g \,\mathbf{e}_{\text{green}} + \alpha_b \,\mathbf{e}_{\text{blue}}\]

Each pixel is a 3-vector of colour/band intensities.

After Conv+ReLU+Pool: low-level features
\[\mathbf{v}_{i,j}^{(1)} = \alpha_1 \,\mathbf{e}_{\text{vert. edge}} + \alpha_2 \,\mathbf{e}_{\text{green area}} + \alpha_3 \,\mathbf{e}_{\text{horiz. gradient}} + \cdots\]

16 learned feature channels: edges, gradients, textures.

Deeper layers: higher-level features
\[\mathbf{v}_{i,j}^{(2)} = \alpha_1 \,\mathbf{e}_{\text{arc}} + \alpha_2 \,\mathbf{e}_{\text{ring frag.}} + \alpha_3 \,\mathbf{e}_{\text{substructure}} + \cdots\]

32 channels: compositions of edges become shapes and lensing-relevant patterns.

Output: a learned summary \(s_\psi(x)\)
\[\mathbf{s}_\psi(x) \in \mathbb{R}^{d_s}\]

A compact feature vector that the SBI head \(q_\phi(\theta\mid s_\psi(x))\) takes as input.

Putting It Together — Convolutional Neural Networks

A convolutional network is generically a composition of convolutional layers (CONV), pooling layers (POOL), linear rectifiers (RELU) and fully connected layers (FC).

A Concrete CNN — LeNet-5

import torch.nn as nn, torch.nn.functional as F
class LeNet5(nn.Module):
def __init__(self):
super().__init__()
self.conv1 = nn.Conv2d(1, 6, 5) # 156 params
self.conv2 = nn.Conv2d(6, 16, 5) # 2,416 params
self.fc1 = nn.Linear(16*4*4, 120) # 30,840
self.fc2 = nn.Linear(120, 84) # 10,164
self.fc3 = nn.Linear(84, 10) # 850
def forward(self, x): # x: 1×28×28
x = F.relu(self.conv1(x)) # → 6×24×24
x = F.max_pool2d(x, 2) # → 6×12×12
x = F.relu(self.conv2(x)) # → 16×8×8
x = F.max_pool2d(x, 2) # → 16×4×4
x = x.flatten(1) # → 256
x = F.relu(self.fc1(x)) # → 120
x = F.relu(self.fc2(x)) # → 84
x = self.fc3(x) # → 10 (logits)
return x

The canonical conv→pool pattern. Spatial dimensions shrink (28→24→12→8→4) while the number of channels grows (1→6→16).

Where the parameters live: ~44k total, but only ~6% are in the convolutional layers. The FC head holds most of the weights, the conv part does the heavy semantic lifting.

For SBI on a lensing image: swap the input shape (e.g. 3×128×128), drop the final fc3, and the remaining feature vector becomes \(s_\psi(x)\). The rest is identical.

LeCun et al., Gradient-Based Learning Applied to Document Recognition, Proc. IEEE 1998.

CNN Architectures

The most common CNN architecture follows the pattern

INPUT → [[CONV → RELU]*N → POOL?]*M → [FC → RELU]*K → FC

  • * indicates repetition, POOL? an optional pooling layer, FC a fully connected (nn.Linear) layer.
  • \(N \geq 0\) (usually \(N \leq 3\)), \(M \geq 0\), \(K \geq 0\) (usually \(K \leq 2\)).
  • The last fully connected layer holds the output (class scores, regression head, or a feature vector for SBI).

Examples:

  • INPUT → FC   (linear classifier)
  • INPUT → [CONV → RELU → POOL]*2 → FC → RELU → FC   (LeNet-style)
  • INPUT → [[CONV → RELU]*2 → POOL]*3 → [FC → RELU]*2 → FC   (VGG-style)

CNNs as Embedding Networks for SBI

The picture: the CNN is the summary statistic \(s_\psi(x)\) in our SBI pipeline.

\(x \xrightarrow{\;\text{CNN}\;s_\psi\;} s \in \mathbb{R}^{d_s} \xrightarrow{\;q_\phi\;} q_\phi(\theta\mid s)\)

Joint training: the CNN parameters \(\psi\) and the density-head parameters \(\phi\) are learned together by minimising

\(\mathcal{L}(\psi, \phi) = -\sum_n \log q_\phi\bigl(\theta_n \,\big|\, s_\psi(x_n)\bigr)\)

on simulator-produced pairs \((\theta_n, x_n)\). No hand-crafted features.

Why this is the right move for lensing: arcs, rings, and substructure are spatial patterns. The CNN's locality, weight sharing, and translation equivariance encode exactly the right inductive bias.

Take-home:

  • CNNs trade resolution for semantics: pixels \(\rightarrow\) feature vector.
  • The whole CNN+density-head stack is one differentiable function trained end-to-end.
  • For any image-shaped observation in physics (lensing, gamma-ray maps, CMB patches, ...), the same recipe applies.
