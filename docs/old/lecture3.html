Logistic Regression & Gradient Descent

From Bayes’ rule to a trainable classifier.

Recap: Regression Predicts Distributions

In Lecture 1: a model \(y(x, \mathbf{w}) = \mathbf{w}^T\boldsymbol{\phi}(x)\) with Gaussian noise predicts a probability distribution:

\[ p(t \mid x, \mathbf{w}, \sigma) = \mathcal{N}\!\left(t \;\middle|\; \mathbf{w}^T\boldsymbol{\phi}(x),\; \sigma^2\right) \]

The output is continuous and unbounded. But what if the target is a class?

What If the Output Is a Class?

Many problems require predicting a category, not a number. Here are three scenarios:

(a) Two classes, linearly separable
(b) Two classes, spiral pattern
(c) Three classes, overlapping

We still want a probability: “how confident is the model that this event is signal?”

The Generative View

Classification data comes from a generative process:

  1. Pick class priors: \(\pi_1 = p(y\!=\!1)\),   \(\pi_0 = 1 - \pi_1\)  
  2. Pick class-conditional distributions \(p(\mathbf{x} \mid y\!=\!0)\) and \(p(\mathbf{x} \mid y\!=\!1)\)
  3. Draw \(N\) samples: for each, pick \(y \sim \text{Bernoulli}(\pi_1)\), then \(\mathbf{x} \sim p(\mathbf{x} \mid y)\)
Given the samples \(\{(\mathbf{x}_n, y_n)\}\), can we learn to predict \(y\) for a new \(\mathbf{x}\)?

From Bayes’ Rule to the Sigmoid

We want \(p(y\!=\!1 \mid \mathbf{x})\). Bayes’ rule gives:

\[ p(y\!=\!1 \mid \mathbf{x}) = \frac{\pi_1 \, p(\mathbf{x} \mid y\!=\!1)}{\pi_0 \, p(\mathbf{x} \mid y\!=\!0) \;+\; \pi_1 \, p(\mathbf{x} \mid y\!=\!1)} \]

Divide numerator and denominator by the numerator:

\[ p(y\!=\!1 \mid \mathbf{x}) = \frac{1}{1 + \frac{\pi_0 \, p(\mathbf{x} \mid y\!=\!0)}{\pi_1 \, p(\mathbf{x} \mid y\!=\!1)}} = \frac{1}{1 + e^{-a}} = \sigma(a) \]

where the logit is the log-odds ratio:   \(\displaystyle a(\mathbf{x}) = \ln \frac{\pi_1 \, p(\mathbf{x} \mid y\!=\!1)}{\pi_0 \, p(\mathbf{x} \mid y\!=\!0)}\)

\(a > 0\): evidence favours class 1   \(a < 0\): evidence favours class 0   \(a = 0\): equal odds

The Logit in Practice

In practice we rarely know \(p(\mathbf{x} \mid y)\). Instead, we parametrise the logit directly:

\[ a(\mathbf{x}) = \mathbf{w}^T\boldsymbol{\phi}(\mathbf{x}) \quad \Rightarrow \quad p(y\!=\!1 \mid \mathbf{x}) = \sigma\!\big(\mathbf{w}^T\boldsymbol{\phi}(\mathbf{x})\big) \]

This is logistic regression. With a neural net for \(a(\mathbf{x})\), it generalises to any decision boundary.

We now have a model \(p(y \mid \mathbf{x}, \mathbf{w})\). What loss function should we minimise to fit \(\mathbf{w}\)?

Cross-Entropy Loss

In Lecture 1: Gaussian noise \(\Rightarrow\) likelihood \(\Rightarrow\) MSE loss. Same logic here:

Bernoulli likelihood (label \(y_n \in \{0, 1\}\), prediction \(\hat{y}_n = \sigma(\mathbf{w}^T\boldsymbol{\phi}(x_n))\)):

\[ p(y_n \mid x_n) = \begin{cases} \hat{y}_n & \text{if } y_n = 1 \\ 1 - \hat{y}_n & \text{if } y_n = 0 \end{cases} \;=\; \hat{y}_n^{\,y_n}(1 - \hat{y}_n)^{1 - y_n} \]

Negative log-likelihood = cross-entropy loss:

\[ E(\mathbf{w}) = -\sum_{n=1}^{N}\Big[ y_n \ln \hat{y}_n + (1 - y_n) \ln(1 - \hat{y}_n) \Big] \]

Gaussian \(\Rightarrow\) MSE Bernoulli \(\Rightarrow\) Cross-entropy
Every loss function has a probabilistic story.

No Closed-Form Solution

Linear regression

Loss is quadratic in \(\mathbf{w}\)

\(\nabla E = 0\) → closed-form solution

\(\mathbf{w}_{\text{ML}} = (\boldsymbol{\Phi}^T\boldsymbol{\Phi})^{-1}\boldsymbol{\Phi}^T\mathbf{t}\)

Logistic regression

Sigmoid makes the loss nonlinear in \(\mathbf{w}\)

No closed-form solution!

\(\nabla E = 0\) → \(\mathbf{w} = \;?\)

In situations where we cannot find a closed-form solution, we need a numerical method to find the minimum of \(E(\mathbf{w})\). The most common one is (stochastic) gradient descent.

Gradient Descent

Start somewhere, then repeatedly step in the direction that decreases the loss:

\[ \mathbf{w} \;\leftarrow\; \mathbf{w} - \eta \,\nabla E(\mathbf{w}) \]

\(\eta\) = learning rate — controls the step size.

Click Next ▶ to see how the learning rate affects convergence.

Gradient Descent — Details

  • The loss \(E(\mathbf{w})\) measures how far the model predictions are from the targets (e.g. MSE or cross-entropy).
  • The gradient \(\nabla E(\mathbf{w})\) points in the direction of steepest increase. We step in the opposite direction.
  • The learning rate \(\eta\) controls step size: too small → slow convergence, too large → overshooting or divergence.
  • The optimal \(\eta\) depends on the curvature of the loss landscape (next slide).
  • GD uses all \(N\) training samples per step. For large datasets this is expensive → mini-batch SGD.

Gradient Descent — Pseudocode

// Given: training data {(xn, tn)}, loss function E, learning rate η

initialise w randomly

for step = 1, 2, ... until converged:
    // Compute gradient over all N samples
    g ← (1/N) ∑n=1N ∇w En(w)

    // Update weights
    w ← w - η · g

Illustration: Step Size and Curvature

How large should \(\eta\) be? Consider a simplistic quadratic loss \(E(w) = \tfrac{1}{2}\, \kappa\, w^2\) with curvature \(\kappa = E''(w)\).

One GD step: \(w_{i+1} = w_i - \eta\, \kappa\, w_i = (1 - \eta\, \kappa)\, w_i\).   Converges instantly when:

\[ \eta = \frac{1}{\kappa} = \frac{1}{E''(w)} \qquad \text{(ideal for this toy case)} \]

Practical challenges: The optimal step size is related to the inverse curvature of the loss, which is typically not known. Furthermore, in many dimensions, curvature differs per direction.

Mini-Batch Stochastic Gradient Descent (SGD)

Computing the full gradient over all \(N\) samples is expensive. Instead, approximate it with a random mini-batch \(\mathcal{B}\) of size \(B \ll N\):

\[ \nabla E(\mathbf{w}) = \frac{1}{N}\sum_{n=1}^{N} \nabla E_n(\mathbf{w}) \;\approx\; \frac{1}{B}\sum_{n \in \mathcal{B}} \nabla E_n(\mathbf{w}) \]

One full pass through the data = one epoch.   Typical batch sizes: \(B\) = 32–256.

When people say “SGD” they almost always mean mini-batch SGD.

SGD = True Gradient + Noise

The mini-batch gradient is a noisy estimate of the true gradient:

\[ \underbrace{\frac{1}{B}\sum_{n \in \mathcal{B}} \nabla E_n}_{\text{mini-batch gradient}} = \underbrace{\nabla E(\mathbf{w})}_{\text{true gradient}} + \underbrace{\boldsymbol{\epsilon}}_{\text{noise}}, \qquad \mathrm{Var}(\boldsymbol{\epsilon}) = \frac{\sigma^2_{\nabla}}{B} \]

\(\sigma^2_{\nabla}\) = variance of individual sample gradients \(\nabla E_n\) across the dataset

Large \(B\)
Low noise, precise gradient
Expensive per step
Small \(B\)
High noise, rough gradient
Cheap per step, helps generalisation

Logistic Regression in Action

Binary classification with hand-picked basis functions \(\boldsymbol{\phi}(\mathbf{x})\). Can they solve the spiral?

  • Linear basis solves the linearly separable case — but fails on the spiral.
  • Higher-order polynomials or RBFs can help, but choosing \(\boldsymbol{\phi}\) by hand is fragile.
  • What if we could learn the basis functions from the data?

Logistic Regression — Details

  • Model: \(p(y\!=\!1 \mid \mathbf{x}) = \sigma(\mathbf{w}^T\boldsymbol{\phi}(\mathbf{x}))\), where \(\boldsymbol{\phi}\) are hand-chosen basis functions (linear, polynomial, or Gaussian RBF).
  • Loss: cross-entropy \(E = -\frac{1}{N}\sum_n [y_n \ln \hat{y}_n + (1-y_n)\ln(1-\hat{y}_n)]\). The gradient w.r.t. \(\mathbf{w}\) has the simple form \((\hat{y}_n - y_n)\,\boldsymbol{\phi}(\mathbf{x}_n)\).
  • Training: mini-batch SGD with batch size \(B\). Each step picks \(B\) random samples, computes the average gradient, and updates \(\mathbf{w}\).
  • Decision boundary: the contour where \(\sigma(\mathbf{w}^T\boldsymbol{\phi}) = 0.5\), i.e. where the logit is zero. Its shape is entirely determined by \(\boldsymbol{\phi}\).
  • Limitation: the quality of the fit depends on the choice of basis functions — a linear basis cannot solve a non-linear problem.

Logistic Regression — Pseudocode

// Given: data {(xn, yn)}, basis φ, batch size B, learning rate η

initialise w = 0

for step = 1, 2, ... :
    // Draw random mini-batch of B samples
    sample B indices from {1, ..., N}

    // Compute predictions and gradient
    for each (xn, yn) in mini-batch:
        an ← wT φ(xn)                    // logit
        ŷn ← σ(an) = 1/(1+e-an)        // predicted probability

    g ← (1/B) ∑n∈B (ŷn - yn) φ(xn)    // gradient of CE loss
    w ← w - η · g                        // SGD update

Learning the Basis Functions

In Lecture 1 we chose \(\boldsymbol{\phi}\) by hand.
What if we let the model learn them?

The \(\boldsymbol{\phi}\) Problem

Everything so far depends on hand-picked basis functions \(\boldsymbol{\phi}(x)\).

Worked for 1D
We could visualise and choose polynomial, Gaussian, or sigmoidal templates.
Doesn't scale
What are good basis functions for images? For particle jets? For spectra in 1000 dimensions?

This limits both regression and classification — the bottleneck is always \(\boldsymbol{\phi}\).

\[ \underbrace{\text{output}}_{\text{prediction}} = \text{head}\!\Big( \underbrace{\mathbf{w}^T \boldsymbol{\phi}(x)}_{\text{this is the bottleneck}} \Big) \]

The head (identity or sigmoid) is trivial to swap. The hard part is \(\boldsymbol{\phi}\).

Towards Learnable Building Blocks

Idea: Replace fixed templates with parametric functions. A simple building block:

\[ \phi_j(\mathbf{x}) = g\!\left(\mathbf{v}_j^T \mathbf{x} + b_j\right) \]

\(g\) = nonlinear activation function — \(\mathbf{v}_j, b_j\) are learnable parameters

For a 1D input, what does this look like with random \(\mathbf{v}, b\)?

Each coloured curve is one learnable basis function \(\phi_j(\mathbf{x}) = g(\mathbf{v}_j^T \mathbf{x} + b_j)\) with random \(\mathbf{v}_j, b_j\). Here: 1D input.

Learnable Building Blocks — Details

  • Each neuron computes \(\phi_j(\mathbf{x}) = g(\mathbf{v}_j^T\mathbf{x} + b_j)\): an affine transformation followed by a nonlinearity.
  • Parameters \(\mathbf{v}_j, b_j\) are learnable: \(\mathbf{v}_j\) controls which direction in input space the neuron responds to, \(b_j\) sets the activation threshold.
  • In 1D: each ReLU neuron is a shifted ramp (a “kink”). A sigmoid neuron is a shifted step-like curve. Their weighted sum can approximate any shape.
  • In higher dimensions: each neuron defines a hyperplane \(\mathbf{v}_j^T\mathbf{x} + b_j = 0\) and activates on one side of it.

Learnable Building Blocks — Pseudocode

// One learnable basis function (neuron)

φj(x) = g(vj · x + bj)

// where:
//   vj = weight (direction in input space)
//   bj = bias (activation threshold)
//   g  = activation function (ReLU, sigmoid, ...)

// H neurons together form a learnable basis:
φ(x) = [φ1(x), φ2(x), ..., φH(x)]

// In matrix form:
φ(x) = g(V x + b)    // V ∈ RH×d, b ∈ RH

One Hidden Layer = Learnable \(\boldsymbol{\phi}\)

Stack \(H\) of these neurons and let a final linear layer combine them:

\[ f(\mathbf{x}) = \sum_{j=1}^{H} w_j \;\underbrace{g(\mathbf{v}_j^T \mathbf{x} + b_j)}_{\text{learnable }\phi_j(\mathbf{x})} \;+\; w_0 \]

This is exactly the Lecture 1 model \(f(\mathbf{x}) = \mathbf{w}^T\boldsymbol{\phi}(\mathbf{x})\), except now the basis functions have learnable parameters.

Input
\(\mathbf{x}\)
→
Hidden layer
\(h_j = g(\mathbf{v}_j^T \mathbf{x} + b_j)\)
→
Output
linear or sigmoid head

This is a neural network with one hidden layer. Stack more layers → a multi-layer perceptron (MLP).

One Hidden Layer — Details

  • Architecture: input \(\mathbf{x} \in \mathbb{R}^d\) → hidden layer with \(H\) neurons → output head. Total parameters: \(H(d+1) + (H+1)\).
  • Hidden layer output: \(\mathbf{h} = g(W^{(1)}\mathbf{x} + \mathbf{b}^{(1)})\), where \(W^{(1)} \in \mathbb{R}^{H \times d}\), \(\mathbf{b}^{(1)} \in \mathbb{R}^H\).
  • Output: \(f(\mathbf{x}) = \mathbf{w}^{(2)T}\mathbf{h} + b^{(2)}\) (regression) or \(\sigma(\mathbf{w}^{(2)T}\mathbf{h} + b^{(2)})\) (classification).
  • Key insight: this is exactly the Lecture 1 model \(\mathbf{w}^T\boldsymbol{\phi}(\mathbf{x})\), but now \(\boldsymbol{\phi}\) has its own learnable parameters.
  • All parameters (\(W^{(1)}, \mathbf{b}^{(1)}, \mathbf{w}^{(2)}, b^{(2)}\)) are trained jointly by SGD on the same loss function.

One Hidden Layer — Pseudocode

// One-hidden-layer neural network
// Input: x ∈ Rd, Output: scalar

// Forward pass:
h = g(W(1) x + b(1))        // hidden layer: Rd → RH
f = w(2)T h + b(2)            // output layer: RH → R
ŷ = head(f)                   // identity (regression) or σ (classification)

// Parameters:
//   W(1) ∈ RH×d   (hidden weights)
//   b(1) ∈ RH       (hidden biases)
//   w(2) ∈ RH       (output weights)
//   b(2) ∈ R         (output bias)
//   Total: H(d+1) + (H+1) parameters

Universal Approximation Theorem

With enough neurons, a single hidden layer can approximate any continuous function.

Intuition: each ReLU neuron contributes a "kink". Combine enough kinks → any shape.

Grey: individual neurons \(w_j \cdot g(v_j x + b_j)\). Black: their sum. More neurons = more expressive.

Universal Approximation — Details

  • Theorem (Cybenko 1989, Hornik 1991): a single hidden layer with any “squashing” activation can approximate any continuous function on a compact domain to arbitrary precision, given enough neurons \(H\).
  • ReLU version: with \(H\) ReLU neurons, the network produces a piecewise-linear function with up to \(H\) “kinks”. As \(H \to \infty\), it can approximate any continuous target.
  • Caveat: the theorem says such weights exist, not that SGD will find them. Training can get stuck in bad local minima, especially for shallow networks.
  • Practical implication: width alone is sufficient in theory, but in practice depth is much more parameter-efficient (next section).

MLP in Action: Regression

Train a one-hidden-layer MLP on 1D regression. Watch the fit evolve:

  • The hidden layer learns its own basis functions — no manual feature engineering.
  • Width controls expressiveness: too few neurons underfit, too many can overfit.

MLP Regression — Details

  • Architecture: \([1, H, 1]\) — 1D input, \(H\) hidden neurons with activation \(g\), linear output (no activation on the head).
  • Loss: mean squared error \(\text{MSE} = \frac{1}{N}\sum_n (\hat{y}_n - t_n)^2\). Evaluated on the full dataset for stable monitoring.
  • Training: mini-batch SGD with \(B = 64\) random samples per step. Each sample does a forward pass and a backward pass (weight update).
  • Fit plot: shows the network’s current function \(f(x)\) vs the target. With ReLU you see piecewise-linear kinks; with sigmoid, smooth curves.
  • Grey loss curve: the mini-batch loss (noisy). Red curve: full-dataset loss (smooth). The gap shows SGD stochasticity.

MLP Regression — Pseudocode

// Given: data {(xn, tn)}, width H, activation g, lr η, batch size B

initialise W(1) ∈ RH×1, b(1) ∈ RH, w(2) ∈ RH, b(2) ∈ R randomly

for step = 1, 2, ... :
    sample B indices from {1, ..., N}

    for each (xn, tn) in mini-batch:
        // Forward pass
        hn ← g(W(1) xn + b(1))          // hidden: R → RH
        ŷn ← w(2)T hn + b(2)             // output: RH → R (no activation)

        // Backward pass + update
        Ln = ½(ŷn - tn)²
        update W(1), b(1), w(2), b(2) via ∇Ln

MLP in Action: Classification

The same architecture, now with a sigmoid head and cross-entropy loss on 2D data:

  • Same network, different head: swap output activation and loss to switch between regression and classification.
  • Complex structured features like the spiral remain challenging to learn with single hidden layer networks.

MLP Classification — Details

  • Architecture: \([2, H, 1]\) — 2D input, \(H\) hidden neurons, sigmoid output. Identical to regression except the head.
  • Loss: cross-entropy \(E = -\frac{1}{N}\sum_n [y_n \ln \hat{y}_n + (1-y_n)\ln(1-\hat{y}_n)]\). Evaluated on full dataset (red curve) and mini-batch (grey curve).
  • Decision boundary: the contour where \(\hat{y} = 0.5\). Contour lines at 10%–90% show the network’s confidence. Blue-orange colorscale is colorblind-safe.
  • Same network, different head: swapping \(\sigma\) for identity and CE for MSE turns this into regression. The hidden representation is shared.
  • Try: circles (width 4 suffices), moons (need ~8), spirals (need ≥16). Depth helps more than width for spirals (section 3).

MLP Classification — Pseudocode

// Given: data {(xn, yn)}, width H, activation g, lr η, batch size B=64

initialise W(1) ∈ RH×2, b(1) ∈ RH, w(2) ∈ RH, b(2) ∈ R randomly

for step = 1, 2, ... :
    sample B indices from {1, ..., N}

    for each (xn, yn) in mini-batch:
        // Forward pass
        hn ← g(W(1) xn + b(1))          // hidden: R2 → RH
        an ← w(2)T hn + b(2)             // logit: RH → R
        ŷn ← σ(an)                      // probability via sigmoid

        // Backward pass + update
        Ln = -[yn ln ŷn + (1-yn) ln(1-ŷn)]   // CE loss
        update W(1), b(1), w(2), b(2) via ∇Ln

Going Deep

One hidden layer is a universal approximator.
But what happens when we stack more?

Stacking Layers

Apply the same idea repeatedly: the output of one layer becomes the input to the next.

→

\(\mathbf{h}^{(1)}\!=\!g(W^{(1)}\mathbf{x}\!+\!\mathbf{b}^{(1)})\)
→

\(\mathbf{h}^{(2)}\!=\!g(W^{(2)}\mathbf{h}^{(1)}\!+\!\mathbf{b}^{(2)})\)

From the output head's perspective, the last hidden layer provides effective basis functions.

With more layers, these effective basis functions become richer and more complex.

How Do We Train Deep Networks?

We need \(\nabla_{\mathbf{W}^{(l)}} L\) for every layer. The chain rule propagates gradients backwards:

\[ \frac{\partial L}{\partial W^{(l)}} = \frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial \mathbf{h}^{(L)}} \cdot \frac{\partial \mathbf{h}^{(L)}}{\partial \mathbf{h}^{(L-1)}} \cdots \frac{\partial \mathbf{h}^{(l+1)}}{\partial \mathbf{h}^{(l)}} \cdot \frac{\partial \mathbf{h}^{(l)}}{\partial W^{(l)}} \]

Each factor \(\partial \mathbf{h}^{(l+1)} / \partial \mathbf{h}^{(l)}\) involves the activation derivative \(g'(\cdot)\) and the weight matrix \(W^{(l+1)}\).

Forward pass
Compute \(\mathbf{h}^{(1)} \to \mathbf{h}^{(2)} \to \cdots \to \hat{y}\)
Backward pass
Propagate \(\nabla L\) from output back to each \(W^{(l)}\)
This is backpropagation — just the chain rule applied systematically.
PyTorch does this automatically via loss.backward(). Details in Lecture 4.

Deep MLP in Action

Now with multiple hidden layers — compare depth and activation function:

  • Depth creates hierarchy: early layers learn simple features, later layers compose them.
  • Computing gradients through many layers is backpropagation — the topic of Lecture 4.

Deep MLP — Details

  • Architecture: \([2, H, H, \ldots, H, 1]\) with \(L\) hidden layers of width \(H\). Total parameters grow as \(\sim L \cdot H^2\).
  • Depth vs width: deeper networks compose features hierarchically — early layers detect simple patterns, later layers combine them. This is more parameter-efficient than a single wide layer.
  • Activation matters: sigmoid can cause vanishing gradients in deep networks (\(\sigma' \leq 0.25\) per layer). ReLU avoids this (\(g' \in \{0, 1\}\)). Details in Lecture 4.
  • Training: mini-batch SGD with \(B = 64\). Gradients through all \(L\) layers are computed by backpropagation (chain rule, automated by PyTorch).
  • Try: spirals with 1 vs 4 layers. One layer struggles; 3–4 layers solve it easily with the same total width.

Deep MLP — Pseudocode

// Given: data {(xn, yn)}, L layers of width H, activation g, lr η, B=64

initialise W(1), b(1), ..., W(L), b(L), wout, bout randomly

for step = 1, 2, ... :
    sample B indices from {1, ..., N}

    for each (xn, yn) in mini-batch:
        // Forward pass through L hidden layers
        h(0) ← xn
        for l = 1, ..., L:
            h(l) ← g(W(l) h(l-1) + b(l))    // RH → RH

        // Output head
        an ← wout,T h(L) + bout            // RH → R
        ŷn ← head(an)                     // σ (classification) or identity (regression)

        // Backward pass (backpropagation through all L+1 layers)
        Ln = loss(ŷn, yn)                // CE or MSE
        update all W(l), b(l), wout, bout via ∇Ln

Key Takeaways

  • Classification = regression with a sigmoid head and cross-entropy loss. Same framework, different output.
  • No closed form → gradient descent. Mini-batch SGD: fast, noisy (noise helps!).
  • The real bottleneck was always the basis functions \(\boldsymbol{\phi}\). Hand-picking them doesn't scale.
  • Solution: make \(\boldsymbol{\phi}\) learnable. \(g(\mathbf{v}^T x + b)\) is the building block. This is a neural network.
  • Depth matters: stacking layers creates richer effective basis functions.
