Bayesian interpretation
of classification networks

The softmax output of a trained CNN is a posterior. Nothing more, nothing less.

What does a CNN classifier actually learn?

Recall Lecture 5: a CNN maps an image to \(K\) class probabilities.

\(\mathbf{x}\)
image
→
CNN \(f_\phi\)
logits \(\in \mathbb{R}^K\)
→
softmax
\(q_\phi(k\mid\mathbf{x})\)
→
CE loss
−log \(q_\phi(k^\star\mid\mathbf{x})\)
\[ q_\phi(k \mid \mathbf{x}) \;=\; \frac{\exp f_\phi(\mathbf{x})_k}{\sum_{j} \exp f_\phi(\mathbf{x})_j} \qquad\qquad \mathcal{L}(\phi) \;=\; -\frac{1}{N}\sum_{i} \log q_\phi(k^{(i)} \mid \mathbf{x}^{(i)}) \]

The CE loss is just maximum likelihood: maximise \(\prod_i q_\phi(k^{(i)}\mid\mathbf{x}^{(i)})\) — the probability of all observed labels given their inputs — which is the same as minimising \(-\!\sum_i \log q_\phi(k^{(i)}\mid\mathbf{x}^{(i)})\).

Claim: at the minimum of \(\mathcal{L}\), the softmax output is a Bayesian posterior.

Training data is a distribution

The training set \(\mathcal{D} = \{(\mathbf{x}^{(i)}, k^{(i)})\}_{i=1}^{N}\) defines an empirical joint distribution:

\[ \hat p(\mathbf{x}, k) \;=\; \frac{1}{N}\sum_{i=1}^{N}\, \delta(\mathbf{x}-\mathbf{x}^{(i)})\, \mathbb{1}[k = k^{(i)}] \]

Which factorises into class prior and class-conditional:

\[ \hat p(\mathbf{x}, k) \;=\; \hat p(k)\,\hat p(\mathbf{x}\mid k), \qquad \hat p(k) = \frac{N_k}{N}, \qquad \hat p(\mathbf{x}\mid k) \;=\; \frac{1}{N_k}\sum_{i:\, k^{(i)}=k} \delta(\mathbf{x}-\mathbf{x}^{(i)}) \]

The dataset is a sample from an underlying data-generating process \(p_\text{true}(\mathbf{x}, k)\). For image classification that process may be:

(i) decide what to photograph (class frequencies), (ii) pick a scene / lighting / camera / angle, (iii) take and keep the photo, (iv) upload to a platform, (v) dataset creator scrapes and filters, (vi) crowd-workers label. Every step injects correlations and biases into \(p_\text{true}\).

In the \(N\to\infty\) idealisation, \(\hat p \to p_\text{true}\) — the delta-combs smooth into a continuous density. This limit is mathematical / conceptual; in practice we always work with finite samples.

Infinite-data limit — minimising CE gives Bayes

Take two idealisations: (i) \(N\to\infty\), so \(\hat p \to p_\text{true}\); (ii) the network is infinitely flexible, so \(q(k\mid\mathbf{x})\) is any function with \(\sum_k q(k\mid\mathbf{x}) = 1\).

\[ \mathcal{L}[q] \;\longrightarrow\; -\!\int\!\sum_{k} p_\text{true}(\mathbf{x}, k)\, \log q(k\mid\mathbf{x})\, d\mathbf{x} \]

Functional derivative w.r.t. \(q(k\mid\mathbf{x})\), under the constraint \(\sum_j q(j\mid\mathbf{x})=1\):

\[ \left.\frac{\delta \mathcal{L}[q]}{\delta q(k\mid\mathbf{x})}\right|_{\sum_j q(j\mid\mathbf{x})\,=\,1} \;=\; 0 \]

\[ \Longrightarrow\quad q^\star(k\mid\mathbf{x}) \;=\; \frac{p_\text{true}(\mathbf{x}, k)}{\sum_j p_\text{true}(\mathbf{x}, j)} \;=\; p_\text{true}(k\mid\mathbf{x}) \]

The softmax output is the Bayesian posterior under \(p_\text{true}\). A finite network just interpolates this idealised answer.

Infinite-data limit — two proofs

Starting from \(\mathcal{L}[q] = -\!\int\!\sum_{k} p_\text{true}(\mathbf{x}, k)\, \log q(k\mid\mathbf{x})\, d\mathbf{x}\), we want the minimiser \(q^\star\).

(a) Lagrange multiplier. Add \(\int\!\lambda(\mathbf{x})(\sum_j q(j\mid\mathbf{x})-1)\,d\mathbf{x}\) to \(\mathcal{L}\).

\[ -\frac{p_\text{true}(\mathbf{x},k)}{q(k\mid\mathbf{x})} + \lambda(\mathbf{x}) = 0 \]

\(\Rightarrow q^\star(k\mid\mathbf{x}) \propto p_\text{true}(\mathbf{x},k)\). Normalising over \(k\) fixes \(\lambda(\mathbf{x}) = p_\text{true}(\mathbf{x})\) and gives

\[ q^\star(k\mid\mathbf{x}) = p_\text{true}(k\mid\mathbf{x}). \]

(b) KL divergence. Rewrite \(\mathcal{L}\) by conditioning on \(\mathbf{x}\):

\[ \mathcal{L}[q] = \mathbb{E}_{\mathbf{x}}\!\big[\mathrm{KL}\!\left(p_\text{true}(\cdot\mid\mathbf{x})\,\|\,q(\cdot\mid\mathbf{x})\right)\big] + H(k\mid\mathbf{x}) \]

The entropy term does not depend on \(q\). KL \(\ge 0\) with equality iff \(q = p_\text{true}(\cdot\mid\mathbf{x})\), so

\[ q^\star(k\mid\mathbf{x}) = p_\text{true}(k\mid\mathbf{x}). \]

Same answer, two lenses: (a) is mechanical (stationary point of a constrained functional); (b) is geometric (CE = KL up to a \(q\)-independent constant, so minimising CE = projecting \(p_\text{true}\) onto the model family under KL).

So… classifiers are doing statistics?

We just showed that a humble image classifier, trained with cross-entropy, converges to \(p_\text{true}(k\mid\mathbf{x})\) — the exact Bayesian posterior for classes given an image.

Image classification is a toy statistical problem: classify pixels into "cat" vs "dog".
But the machinery — sample pairs, minimise the NLL \(\bigl(\text{negative log-likelihood, } -\!\sum_i \log q_\phi(k^{(i)}\!\mid\!\mathbf{x}^{(i)})\bigr)\), read off a posterior — is fully general.

The question for today: can we use the same machinery for the statistical problems we actually care about in physics — inferring cosmological parameters from surveys, black-hole masses from gravitational waves, dark-matter substructure from lensing images?

Simulation-based inference
and NPE

Training pairs \((\boldsymbol\theta, \mathbf{x}) \sim p(\boldsymbol\theta)\,p(\mathbf{x}\mid\boldsymbol\theta)\) from a simulator — minimise \(-\!\log q_\phi(\boldsymbol\theta\mid\mathbf{x})\) — read off the posterior.

Simulation-based inference. We have only the forward direction (a simulator). SBI builds the inverse by drawing \((\boldsymbol\theta, \mathbf{x})\) pairs from the simulator and training a neural network to output the posterior directly — just like the CNN classifier did for images.

The inference problem — formally

Same story, written with physics notation. Parameters \(\boldsymbol\theta\) take the role of the class or parameters of interest.

\[ p(\boldsymbol\theta \mid \mathbf{x}) \;=\; \frac{p(\mathbf{x}\mid\boldsymbol\theta)\, p(\boldsymbol\theta)}{p(\mathbf{x})} \]
  • Prior \(p(\boldsymbol\theta)\): what we believed before seeing data.
  • Likelihood \(p(\mathbf{x}\mid\boldsymbol\theta)\): what the physics model predicts.
  • Posterior \(p(\boldsymbol\theta\mid\mathbf{x})\): what we want.
  • Evidence / Marginal likelihood \(p(\mathbf{x}) = \int p(\mathbf{x}\mid\boldsymbol\theta)\, p(\boldsymbol\theta)\, d\boldsymbol\theta\).

Note. SBI only requires samples from \(p(\mathbf{x}\mid\boldsymbol\theta)\) — i.e. a running simulator. Likelihood-based methods (MCMC, HMC, VI) need pointwise evaluations of \(p(\mathbf{x}\mid\boldsymbol\theta)\), which are often unavailable when detector effects or systematics must be marginalised (= integrated) out.

The SBI pipeline — the simulator

Simulator — encodes the physics.
Maps parameters \(\boldsymbol\theta\) to data realisations \(\mathbf{x} \sim p(\mathbf{x}\mid\boldsymbol\theta)\).

Examples:

  • Strong-lensing image generators (source + lens model + PSF + noise).
  • Gravitational-wave waveform models (LALSuite, IMRPhenom).
  • Cosmological N-body / hydro (Quijote, CAMELS).
  • Particle-physics event generators (Pythia, MadGraph).

⚠ Failure mode: wrong model → wrong inference. Misspecification of the simulator propagates into the posterior, often invisibly.

The SBI pipeline — the inference loss

Inference loss \(\mathcal{L}\) — defines the statistical question.
The choice of loss is what turns the network into a particular kind of estimator.

The loss decides what we're solving:

  • Classification → cross-entropy.
  • Parameter regression → (Gaussian) NLL of \(\boldsymbol\theta^\star\).
  • Model discrimination → binary cross-entropy on a ratio.
  • Object detection → localisation + classification compound loss.
  • …

Same simulator, same network, different loss ⇒ different statistical task.

Three example inference heads

All three minimise \(-\log q_\phi(\boldsymbol\theta^\star\mid\mathbf{x})\) on simulator pairs. Only the density family changes.

  • Multi-class classifier. Discretise \(\boldsymbol\theta\) into \(K\) bins; output a softmax over bins. \(\;q_\phi(k\mid\mathbf{x}) = \mathrm{softmax}(f_\phi(\mathbf{x}))_k\). Loss: cross-entropy.
  • Gaussian head. Output \(\boldsymbol\mu_\phi(\mathbf{x})\) and \(\boldsymbol\Sigma_\phi(\mathbf{x})\). \(\;q_\phi(\boldsymbol\theta\mid\mathbf{x}) = \mathcal{N}(\boldsymbol\theta \mid \boldsymbol\mu_\phi, \boldsymbol\Sigma_\phi)\). Loss: Gaussian NLL.
  • Normalising flow. Output a sequence of invertible maps conditioned on \(\mathbf{x}\). \(\;q_\phi(\boldsymbol\theta\mid\mathbf{x})\) arbitrary, evaluated via change of variables. Loss: flow NLL (with Jacobians).

These three are examples of the same principle. Other approaches exist (NLE: learn the likelihood; NRE: learn the likelihood-to-evidence ratio; mixture-density networks; score-based methods; …) and are not covered here.

Example head 1: classification

A trained softmax classifier outputs a posterior over classes given an input.

Training loop 
// Inputs: dataset {(x_i, k_i)}, network f_phi
for step = 1, 2, ... :
    sample mini-batch B from dataset
    for (x, k) in B:
        logits = f_phi(x)              // in R^K
        q      = softmax(logits)
        L     += -log q[k]             // CE loss
    update phi via ∇L

Loss.

\( \mathcal{L}(\phi) = -\frac{1}{N}\sum_{i=1}^{N} \log q_\phi(k^{(i)}\mid\mathbf{x}^{(i)}) \)

Convergence (infinite data, infinitely flexible network).

\( q_\phi^\star(k\mid\mathbf{x}) \;=\; p_\text{true}(k\mid\mathbf{x}) \)

I.e.\ the Bayesian posterior over classes given \(\mathbf{x}\), under the data-generating distribution \(p_\text{true}\).

Example head 2: Gaussian-head NPE

Replace classes by continuous parameters; replace the softmax by a conditional Gaussian.

Training loop 
// Inputs: simulator p(theta), p(x|theta), network f_phi
for step = 1, 2, ... :
    for i = 1, ..., B:
        theta_i ~ p(theta)             // draw from prior
        x_i     ~ p(x | theta_i)       // run simulator
        mu_i, log_sigma_i = f_phi(x_i)
        L      += -log N(theta_i | mu_i, sigma_i^2)
    update phi via ∇L

Density family.

\( q_\phi(\boldsymbol\theta\mid\mathbf{x}) = \mathcal{N}\!\big(\boldsymbol\theta\,\big|\,\boldsymbol\mu_\phi(\mathbf{x}), \boldsymbol\Sigma_\phi(\mathbf{x})\big) \)

Both \(\boldsymbol\mu_\phi(\mathbf{x})\) and \(\boldsymbol\Sigma_\phi(\mathbf{x})\) can depend on the data — the network learns where to be confident and where not to be.

Loss = Gaussian NLL.

\( \mathcal{L} = \tfrac{1}{2}(\boldsymbol\theta-\boldsymbol\mu)^\top\boldsymbol\Sigma^{-1}(\boldsymbol\theta-\boldsymbol\mu) + \tfrac{1}{2}\log\det\boldsymbol\Sigma \)

Constant \(\boldsymbol\Sigma\) reduces to MSE (Lecture 1 callback).

Demo — model details

  • Forward model: \(y_i = a + b\, x_i + \varepsilon_i\), with \(\varepsilon_i \sim \mathcal{N}(0, 0.3^2)\) and \(x_i\) on a fixed 30-point grid in \([0, 1]\).
  • Priors: \(a \sim \mathcal{U}[-1, 1]\), \(b \sim \mathcal{U}[-3, 3]\).
  • Network \(f_\phi\): MLP with layer sizes \([30, 32, 16, 5]\), ReLU activations, linear output. Input is the 30-point observation vector \(\mathbf{y}\); output is \((\mu_a, \mu_b, \log\sigma_a, \log\sigma_b, z)\).
  • Conditional Gaussian with correlation: \(q_\phi(a, b \mid \mathbf{y}) = \mathcal{N}\!\big((a, b) \,\big|\, \boldsymbol\mu, \boldsymbol\Sigma\big)\) with \[ \boldsymbol\Sigma = \begin{pmatrix} \sigma_a^2 & \rho\,\sigma_a\sigma_b \\ \rho\,\sigma_a\sigma_b & \sigma_b^2 \end{pmatrix}, \qquad \rho = \tanh(z) \in (-1, 1). \] The \(\tanh\) ensures \(\rho\) is a valid correlation coefficient, so \(\boldsymbol\Sigma\) stays positive-definite. The posterior is anti-correlated: increasing the slope \(b\) requires lowering the offset \(a\) to fit the same data — the network discovers this without being told.
  • Loss: bivariate Gaussian NLL of \((a^\star, b^\star)\) under \(q_\phi\), averaged over a fresh mini-batch of size 64 drawn from the simulator at every step.
  • Optimiser: Adam (\(\beta_1 = 0.9\), \(\beta_2 = 0.999\)) with learning rate set by the slider.
  • "MCMC" (ground truth): we know \(p(\mathbf{y}\mid a, b)\) analytically here, so the comparison is importance-sampled (20k prior draws, weight \(\propto\) likelihood, resample 1000). For real problems the simulator's likelihood would be intractable — that is why we used SBI in the first place.

The SBI pipeline — the embedding network

Embedding network \(T_\phi(\mathbf{x})\) — data compression.
Maps high-dimensional observations to a low-dimensional summary that the inference head can chew on.

Why we need it:

  • Strong lensing: \(\mathbf{x}\) is a \(256\times 256\) image — \(\sim\!65{,}000\) pixels.
  • Gravitational waves: \(\sim\!16{,}000\) time-series samples per detector.
  • Cosmology: density fields of \(\sim\!10^9\) voxels.

A raw MLP would have too many parameters and no inductive bias. The CNN from Lecture 5 (or Transformer / GNN) does the compression. Trained jointly with the inference head: \(T_\phi\) learns features informative for inference, not for class labels.

Live demo on images — setup

Forward model: a Gaussian blob on a \(32 \times 32\) image, centred at \((c_x, c_y)\), with fixed radius \(\sigma = 0.15\) and i.i.d.\ pixel noise \(\varepsilon_{ij} \sim \mathcal{N}(0, 0.45^2)\).

\[ I_{ij} \;=\; \exp\!\left(-\tfrac{(x_j - c_x)^2 + (y_i - c_y)^2}{2\sigma^2}\right) + \varepsilon_{ij}, \qquad (c_x, c_y) \sim \mathcal{U}\!\big([-0.7, 0.7]^2\big). \]

Rich network (no bottleneck)

MLP [1024 → 256 → 64 → 32 → 2]

The flattened image is fed straight to a wide first hidden layer. Enough capacity to learn implicitly the few linear summaries that locate the blob.

Lossy network (explicit 8-D bottleneck)

MLP [1024 → 8 → 64 → 32 → 2]

Same downstream architecture, but the image is crushed through a linear \(1024 \to 8\) projection up front. The inference head only sees an 8-dimensional summary.

Both networks output means only (the two final outputs are \(\boldsymbol\mu_\phi(I) = (\mu_{c_x}, \mu_{c_y})\)). The posterior \(q_\phi(c_x, c_y \mid I) = \mathcal{N}(\boldsymbol\mu_\phi(I), \boldsymbol\Sigma)\) uses a homoscedastic \(\boldsymbol\Sigma\) — three learnable scalars \((\log\sigma_a, \log\sigma_b, z)\) per network, shared across all images, with \(\rho = \tanh(z)\).

Normalising flows

A flexible density from invertible maps — via the change-of-variables formula.

Why we need flows

Real posteriors in physics are often multimodal, curved, or heavy-tailed. Gaussians can't represent them.
① Evaluate \(\log q_\phi(\boldsymbol\theta\mid\mathbf{x})\)

Needed for the NPE loss.
② Sample \(\boldsymbol\theta \sim q_\phi\)

Needed for corner plots, downstream analyses.
③ Arbitrary shape

Multimodal, curved, heavy-tailed.
MLPs on raw outputs fail ①: nothing forces normalisation.
Normalising flows satisfy all three by construction, via the change-of-variables formula.

Change of variables

1D. If \(z \sim p_Z(z)\) and \(\theta = f(z)\) with \(f\) invertible and differentiable, \[ p_\Theta(\theta) \;=\; p_Z(z)\,\left|\frac{dz}{d\theta}\right| \;=\; p_Z(f^{-1}(\theta))\,\left|\frac{df}{dz}\right|^{-1} \]
Multivariate. For \(\mathbf{z} \in \mathbb{R}^d\), \(\boldsymbol\theta = f(\mathbf{z})\), \[ p_\Theta(\boldsymbol\theta) \;=\; p_Z(f^{-1}(\boldsymbol\theta))\;\left|\det \frac{\partial f}{\partial \mathbf{z}}\right|^{-1} \] The Jacobian determinant is the volume rescaling. General \(|\det J|\) costs \(\mathcal{O}(d^3)\).

Flow design: pick architectures where \(J\) is triangular, so \(|\det J| = \prod_i J_{ii}\) costs \(\mathcal{O}(d)\).

Worked example: warping a Gaussian

map  \(\theta = z + A\sin z\)
base  \(\mathcal{N}(0, 1.5^2)\)
warped  \(p_\Theta(\theta) = p_Z(z)/|1+A\cos z|\)
monotonic ✓

Base \(\mathcal{N}(0,1.5^2)\) pushed through \(\theta = z + A\sin z\). For \(A < 1\) the map is strictly monotonic — still invertible — but as \(A\to 1\) the derivative near \(z=\pm\pi\) collapses, so density mass piles up there: the warped target becomes bimodal.

Normalising flow = stack of invertible maps

\[ \boldsymbol\theta \;=\; f_L \circ f_{L-1} \circ \cdots \circ f_1(\mathbf{z}), \qquad \mathbf{z} \sim \mathcal{N}(0, I_d) \] Each \(f_\ell\) is invertible and differentiable, with parameters \(\phi_\ell\) (possibly conditioned on \(\mathbf{x}\)).
  • Sampling \(\boldsymbol\theta \sim q_\phi\): draw \(\mathbf{z}\), push forward through the stack.
  • Density evaluation at a given \(\boldsymbol\theta\): invert the stack, evaluate \(p_Z\), correct with Jacobians.
  • Composition of invertible maps is invertible — deep flows stay a valid density.
Flexibility grows with depth \(L\), just like MLPs — but every layer preserves the probability axioms.

Log-density by chain of Jacobians

Let \(\mathbf{z}_\ell = f_\ell(\mathbf{z}_{\ell-1})\), \(\mathbf{z}_0 = \mathbf{z}\), \(\mathbf{z}_L = \boldsymbol\theta\). Change-of-variables composes: \[ \log q_\phi(\boldsymbol\theta\mid\mathbf{x}) \;=\; \log p_Z(\mathbf{z}_0) \;-\; \sum_{\ell=1}^{L} \log \left| \det \frac{\partial f_\ell}{\partial \mathbf{z}_{\ell-1}} \right| \]
Training is just minibatch SGD: \[ \mathcal{L}_{\mathrm{NPE}}(\phi) = -\frac{1}{|\mathcal{B}|}\sum_{(\boldsymbol\theta, \mathbf{x})\in\mathcal{B}} \log q_\phi(\boldsymbol\theta\mid\mathbf{x}) \] The whole stack, including any conditioning network, trains end-to-end with backprop.

Key design constraint: each \(|\det \partial f_\ell / \partial \mathbf{z}_{\ell-1}|\) must be cheap.

Affine coupling layers

Split \(\mathbf{z} = (\mathbf{z}_A, \mathbf{z}_B)\). Define \[ \mathbf{z}'_A = \mathbf{z}_A, \qquad \mathbf{z}'_B = \mathbf{z}_B \odot \exp\!\big(s_\phi(\mathbf{z}_A)\big) \;+\; t_\phi(\mathbf{z}_A) \] where \(s_\phi, t_\phi\) are arbitrary neural networks (MLPs).
Invertibility
\(\mathbf{z}_A\) passes through untouched; \(\mathbf{z}_B\) is scaled-and-shifted given \(\mathbf{z}_A\). Trivially invertible in closed form.
Cheap Jacobian
\(J\) is triangular ⇒ \(\log|\det J| = \sum_i s_\phi(\mathbf{z}_A)_i\). \(\mathcal{O}(d)\), not \(\mathcal{O}(d^3)\).

Alternate which half is "active" across layers (RealNVP). Stack \(L\) of these → expressive flow.

Conditional flows — plugging in the data

For SBI we want \(q_\phi(\boldsymbol\theta \mid \mathbf{x})\), not just \(q_\phi(\boldsymbol\theta)\).
Let the scale and shift networks depend on the data (or its embedding): \[ s_\phi(\mathbf{z}_A, T_\phi(\mathbf{x})), \qquad t_\phi(\mathbf{z}_A, T_\phi(\mathbf{x})) \]
End-to-end picture.
  1. CNN embedding \(T_\phi(\mathbf{x})\) compresses the data.
  2. Conditional flow turns \(\mathbf{z}\sim\mathcal{N}(0,I)\) into \(\boldsymbol\theta\), with all coupling-layer parameters depending on \(T_\phi(\mathbf{x})\).
  3. Loss: \(-\log q_\phi(\boldsymbol\theta^\star\mid\mathbf{x})\) averaged over simulation pairs.
Every component — CNN, coupling networks \(s, t\) — is trained together by backprop.

Flow pushforward — 2D worked example

Base samples \(\mathbf{z}\sim\mathcal{N}(0, I)\) are warped layer-by-layer into the target shape (here: two-moons). Each layer's Jacobian contributes to the log-density, keeping normalisation exact.

In the conditional version, the target shape depends on \(\mathbf{x}\) — each observation gets its own warped posterior.

The full pipeline, revisited

Simulator: physics model, possibly slow.
Embedding \(T_\phi\): CNN / Transformer / GNN — matches the data modality.
Conditional flow: coupling layers conditioned on \(T_\phi(\mathbf{x})\).
Inference loss: \(-\log q_\phi(\boldsymbol\theta^\star\mid\mathbf{x})\).
Amortised: train once, then inference on any new \(\mathbf{x}_{\text{obs}}\) is a single forward pass + sampling.

Application — stochastic GW background with LISA

Posteriors for stochastic-background amplitude \(\alpha\), tilt \(\gamma\), and two noise parameters \(A\), \(P\). Inset: injected PSD — noise + SGWB + unresolved transients.
Problem. LISA data = instrument noise + a stochastic GW background + hundreds of unresolved transient sources (SMBH mergers). Goal: infer the background parameters.
Why SBI? The likelihood over the transient population is intractable — traditional MCMC either ignores transients (biased, orange dashed) or has to sample their full parameter space (prohibitively expensive).
Result. The SBI posterior (orange solid, TMNRE via saqqara) automatically marginalises over the transients and recovers the true injection — matching MCMC (black) on the clean case C1.
Take-away. SBI turns a nuisance (the transient population) into something implicit in the simulator — no likelihood engineering needed.
