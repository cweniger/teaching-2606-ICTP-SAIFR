What Does an LLM Actually Do?

An LLM does one thing: predict the next token.
The cat sat on the ?

mat (62%)  •  floor (18%)  •  table (7%)  •  roof (3%)  •  ...

Why does this work? Predicting text well requires implicitly modelling facts, logic, and reasoning. The raw model is then fine-tuned with human feedback (RLHF) to follow instructions and behave helpfully.

Fluent
Trained on trillions of tokens
Confident when wrong
Optimised for plausible, not correct
Hallucinations
Most likely token ≠ true token
Great at code
Code is highly predictable text

We'll see exactly how this works in the last lecture (transformers, attention, tokenisation).

Using GenAI Effectively

Learning, coding, and research

Learning with GenAI

Good patterns
  • Socratic tutor: "Explain this to me step by step"
  • Explain it back: "Here's my understanding — is this correct?"
  • Challenge it: "Are you sure about step 3? I think …"
  • Multiple angles: "Give me another example" / "Quiz me"
  • Close the chat, re-derive it yourself
Bad patterns
  • Copy-paste the homework question, submit the answer
  • Accept the first answer without checking
  • Memorise the AI's answer instead of understanding the reasoning
  • Skip the derivation because the AI did it
  • Let it do the thinking and decision making so you don't have to

Start with clear questions in mind. If you have none, start with the lecture material.

Four Principles

1. Context is everything

"I'm a physics master student learning about Bayesian inference for the first time. Explain the posterior predictive distribution."

vs. "Explain posterior predictive distribution" — you'll get a very different answer.

2. Be specific

"Derive the MAP estimate for linear regression with a Gaussian prior, step by step, showing where each term comes from."

vs. "Explain MAP estimation" — too vague, you'll get a generic overview.

3. Challenge the answer

"Are you sure about step 3? I think the sign should be different because …"
"What assumptions are you making? Are they valid here?"

4. Iterate

The first answer is rarely the best. Follow up: "Good, but simplify the notation." "Now show me with numbers." "Explain it using a physics analogy."

Code Generation: Understand and Verify

Where it excels
  • Boilerplate and scaffolding
  • Standard algorithms & data wrangling
  • Debugging: "Why does this give me this error?"
  • Translating between languages / frameworks
  • Writing tests
Where to be careful
  • Subtle numerical issues (precision, stability)
  • Domain-specific physics code
  • Complex logic with many edge cases
  • Performance-critical code
  • Code that requires understanding of the full project context
Golden rule: Always read and test generated code. If you can't understand it, you can't trust it.

Factual Claims: The Wikipedia Standard

Treat LLM output like Wikipedia —
useful for orientation, never a primary source.

This depends on the mode. With internet access or tool use, an LLM can generate accurate literature lists and check facts. Without tools, it works from memory — and memory is where things go wrong:

Paper references
Without search: often plausible but invented
Specific numbers
Without computation: may be approximate or wrong
Historical claims
Events, quotes, attributions — often subtly garbled
With tools: verify the source

LLM + search can find real papers. But always check that the link works, the paper exists, and it says what the AI claims.

Without tools: assume nothing

Pure recall = the vanilla LLM from the dinner party. "The AI said so" is not a citation.

When Agents Go Wrong

Case study (2025): An autonomous AI agent submitted a code contribution to matplotlib (a major open-source Python library). When the maintainer rejected it through normal code review, the agent autonomously:
  • Researched the maintainer's personal history online
  • Published a blog post attacking his character and reputation
  • Framed the rejection as discrimination rather than quality control
The risk
Autonomous agents can take actions with real-world consequences that no human intended or approved.
No accountability
No clear person was responsible. The agent ran with minimal oversight on someone's computer.
The lesson
More autonomy = more capability, but also more risk. Human oversight is not optional.

Source: S. Shambaugh, "An AI agent published a hit piece on me" (2025)

GenAI Policy

University framework, course tiers, and what it means for you

UvA Policy on GenAI

UvA Policy Framework & Guidelines on GenAI in Education (2025)

Individual achievement
Student work must represent the student's own achievement. You must be able to explain and justify it.
Assessment responsibility
The examiner retains final responsibility for assessment and grading, irrespective of any GenAI support used.
Transparency & integrity
GenAI use must be transparent and documented. Unreported use that prevents proper assessment may constitute fraud under the OER.
Equality & privacy
Students must be able to pass without GenAI unless stated otherwise. No privacy-sensitive data in non-approved tools.

Experimental GenAI Tier-based System (for this course)

Idea: Courses specify tiers per component. Two student-facing dimensions, two lecturer-facing.

T — Student use

T0 Not permitted
Work produced without GenAI; exams, oral assessments
T1 Limited, assistive
Clarification, debugging, polishing; no generated solutions or final text
T2 Co-pilot with responsibility
May produce text/code; student must verify, understand & disclose
T3 AI-integrated activity
Using, evaluating, or critiquing GenAI is part of the assessment

S — AI skills

S0 No GenAI skill objectives
No aim to develop GenAI skills beyond incidental exposure
S1 Basic AI literacy
Guidance on responsible use, hallucinations, bias; not assessed
S2 Applied GenAI practice
Verification, documentation, reproducibility; may be required in coursework
S3 Critical GenAI competence
Assessed on judging reliability, limitations & impact of GenAI tools

M — Materials

M0 No GenAI use
Materials produced without GenAI assistance
M1 Editing & presentation
Language, formatting, style; lecturer remains the author
M2 Drafting with curation
AI drafts explanations, exercises, code; lecturer reviews & corrects
M3 GenAI as teaching object
AI output included for illustration, critique, or comparison

G — Grading

G0 No GenAI use
All grading and feedback by human examiners
G1 Feedback drafting
AI drafts comments; examiner determines final feedback & grades
G2 Rubric-based support
AI proposes rubric alignment; examiner reviews & confirms
G3 Highly automated (exceptional)
Examiner retains final judgement; low-stakes contexts only

Accountability: Does Your Work Hold Up?

The blackboard test: Could you defend this at the blackboard if someone challenged you?

Being accountable means:

What
You can explain what it does and how it works
Why

Key Takeaways

  • An LLM predicts the next token. Fluent, not necessarily correct.
  • Thinking, tools, and agents add capabilities on top — but you are accountable for the output.
  • Use it as a Socratic tutor and sparring partner, not an oracle.
  • Factual claims follow the Wikipedia standard: useful for orientation, never a primary source.
  • If you can't explain it without the AI, you didn't learn it.
