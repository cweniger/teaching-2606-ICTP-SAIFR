What is Machine Learning?

Machine learning is a specific approach to learn something about data generating processes in nature, whether from real-world observations, simulations, or human behaviour.

It works by optimizing a computer program to capture generalizable characteristics of these processes, based on a finite number of examples or targeted interactions.

Underpinning all of this is probability theory — the natural language for reasoning about uncertainty — just as the laws of nature underpin applied physics.

The Idea: Template functions & Fitting Parameters

Selected basis functions \(\phi_j(x)\) are supposed to encode plausible functional behaviour. We combine \(M\) of them with free parameters:

\[ y(x, \mathbf{w}) = \sum_{j=0}^{M-1} w_j \,\phi_j(x) = \mathbf{w}^T \boldsymbol{\phi}(x), \qquad \mathbf{w} \in \mathbb{R}^M \]
  • Basis functions \(\phi_j(x)\) — fixed templates capturing expected shapes
  • Weight vector \(\mathbf{w} = (w_0, \ldots, w_{M-1})^T\) — \(M\) free parameters, fitted to data
  • "Linear" refers to linearity in \(\mathbf{w}\), not in \(x\) — the templates themselves can be nonlinear!

From Process to Data

Nature picks weights \(\mathbf{w}\), producing a function \(y(x) = \mathbf{w}^T \boldsymbol{\phi}(x)\). You observe noisy samples: \(\;t_n = y(x_n) + \epsilon\), \(\;\epsilon \sim \mathcal{N}(0,\sigma^2)\).

Notation: \(y(x)\) is the true (unknown) function; \(t_n\) are the noisy observed values — what you actually have access to.

Given only the data points — can you recover the black curve?

From Noise to Likelihood

Assume Gaussian noise: \(\;t_n = \mathbf{w}^T \boldsymbol{\phi}(x_n) + \epsilon\), \(\;\epsilon \sim \mathcal{N}(0, \sigma^2)\)

Joint likelihood (assuming independent noise):

\[ p(\mathbf{t} \mid \mathbf{w}, \sigma) \equiv \prod_{n=1}^{N} \mathcal{N}\!\left(t_n \,\middle|\, \mathbf{w}^T\boldsymbol{\phi}(x_n),\, \sigma^2\right) \]

Log-likelihood:

\[ \ln p(\mathbf{t} \mid \mathbf{w}, \sigma^2) = \text{const} - \frac{1}{2\sigma^2} \underbrace{\sum_{n=1}^{N}\left(t_n - \mathbf{w}^T\boldsymbol{\phi}(x_n)\right)^2}_{\equiv\, 2\,E(\mathbf{w})} \]

Maximising the likelihood = minimising the sum-of-squares error \(E(\mathbf{w})\).

The Gaussian Distribution

We just assumed Gaussian noise — let's recall what that means.

\[ \mathcal{N}(z \mid \mu, \sigma^2) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp\!\left(-\frac{(z-\mu)^2}{2\sigma^2}\right) \]
  • Central limit theorem: sum of many independent random variables → Normal
  • Ubiquitous in physics — measurement errors, thermal fluctuations, ...

Solving for the Weights

Loss function:

\[ E(\mathbf{w}) = \frac{1}{2}\sum_{n=1}^{N}\left(t_n - \mathbf{w}^T\boldsymbol{\phi}(x_n)\right)^2 \]

Set \(\nabla E(\mathbf{w}) = 0\). This is convex — one global minimum.

Closed-form solution via the design matrix \(\boldsymbol{\Phi} \in \mathbb{R}^{N \times M}\) (\(\Phi_{nj} = \phi_j(x_n)\)):

\[ \mathbf{w}_{\text{ML}} = \underbrace{\left(\boldsymbol{\Phi}^T\boldsymbol{\Phi}\right)^{-1}\boldsymbol{\Phi}^T}_{M \times N}\,\underbrace{\mathbf{t}}_{N} \]

Three equivalent views: maximise likelihood = minimise loss = solve normal equations.

Not All Models Are Equal

Maximum likelihood finds the \(\mathbf{w}\) that best explains the data — but is \(\mathbf{w}\) itself plausible?

M = 4: \(\mathbf{w} = (0.3,\; -0.5,\; 0.1,\; 0.2)^T\)
M = 13: \(\mathbf{w} = (452,\; -1803,\; 2741,\; \ldots)^T\)

Both explain the training data. But huge weights mean fine-tuned cancellations — fragile, implausible.

"Some parameter values are more plausible than others, before we look at data." → Bayes' theorem.

Beyond Maximum Likelihood

Overfitting showed us that maximum likelihood isn't enough.
Can we do better by encoding prior knowledge?

Bayes' Theorem

\[ \underbrace{p(\mathbf{w} \mid \mathbf{t})}_{\text{posterior}} = \frac{\overbrace{p(\mathbf{t} \mid \mathbf{w})}^{\text{likelihood}} \;\times\; \overbrace{p(\mathbf{w})}^{\text{prior}}}{\underbrace{p(\mathbf{t})}_{\text{evidence}}} \]
Prior \(p(\mathbf{w})\)
What we believe about the parameters before seeing data.
Likelihood \(p(\mathbf{t}|\mathbf{w})\)
How well these parameters explain the data.
Posterior \(p(\mathbf{w}|\mathbf{t})\)
Updated belief after seeing data.

Thomas Bayes (1701–1761)
Portrait attribution uncertain

The posterior balances fitting the data (likelihood) with staying plausible (prior).

The posterior is always pulled towards the prior — especially when data is scarce or uncertain.

From Bayes to Regularisation

Choose a Gaussian prior: \(\; p(\mathbf{w}) = \mathcal{N}(\mathbf{0},\, \sigma_w^2\mathbf{I})\) — "weights are probably small."

MAP estimate (maximum a posteriori) — maximise the log-posterior:

\[ \ln p(\mathbf{w} \mid \mathbf{t}) = \underbrace{-\frac{1}{2\sigma^2}\sum_n\!\left(t_n - \mathbf{w}^T\boldsymbol{\phi}(x_n)\right)^2}_{\text{log-likelihood}} \underbrace{- \frac{1}{2\sigma_w^2}\|\mathbf{w}\|^2}_{\text{log-prior}} + \text{const} \]

Equivalently, define the regularised loss \(\bigl(\lambda = \sigma^2/\sigma_w^2\bigr)\):

\[ \tilde{E}(\mathbf{w}) \equiv E(\mathbf{w}) + \frac{\lambda}{2}\|\mathbf{w}\|^2 \]

The penalty \(\frac{\lambda}{2}\|\mathbf{w}\|^2\) discourages large weights — exactly what we wanted to prevent overfitting.

The Regularised Solution

Setting \(\nabla \tilde{E}(\mathbf{w}) = 0\) gives the MAP estimate:

\[ \mathbf{w}_{\text{MAP}} = \left(\underbrace{\boldsymbol{\Phi}^T\boldsymbol{\Phi} + \lambda\mathbf{I}}_{\text{stabilised by } \lambda\mathbf{I}}\right)^{-1}\boldsymbol{\Phi}^T\,\mathbf{t} \]
\(\lambda \to 0\)
No prior — recovers \(\mathbf{w}_{\text{ML}}\)
Moderate \(\lambda\)
Balances fit and simplicity
\(\lambda \to \infty\)
Prior dominates — \(\mathbf{w} \to \mathbf{0}\)

The \(\lambda\mathbf{I}\) term makes the matrix better conditioned — the solution is stable even when \(M > N\).

Key Takeaways

  • Linear regression = basis functions + weights. "Linear" means linear in the weights, not in \(x\).
  • Probabilistic view: Gaussian noise → likelihood → least-squares loss. Every loss function has a probabilistic story.
  • Overfitting = the model memorises noise. Detect it with validation data, never with training data.
  • Bayes: a prior on the weights leads to regularisation. The penalty \(\lambda\|\mathbf{w}\|^2\) is not ad hoc — it's a modelling choice.
  • We predict distributions, not numbers. Probability theory is the language of machine learning.
