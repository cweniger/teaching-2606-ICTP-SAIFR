Shaping the Training Trajectory

We can bias the trajectory toward generalizable features using regularization techniques:

Weight decay
Penalize large weights → prefer simpler functions
Dropout
Randomly drop neurons → force redundant representations
Batch normalization
Stabilize activations → smoother optimization landscape
Early stopping
Stop before noise is memorized (see previous slide)
Data augmentation
Artificially increase training set → harder to memorize
And more...
Label smoothing, mixup, spectral normalization, ...
Let's look at the first three in detail.

Weight Decay (L2 Regularization)

Callback to Lecture 1: a Gaussian prior on weights gives a penalty \(\lambda\|\mathbf{w}\|^2\).

\[E_{\text{reg}} = E_{\text{data}} + \frac{\lambda}{2}\sum_i w_i^2\] In practice: add \(\lambda\,w_i\) to every gradient   \(\Longrightarrow\)   weights shrink toward zero.
Effect: prevents any single weight from growing too large. Encourages simpler functions.
Probabilistic view: same as MAP estimation with a Gaussian prior. Ridge regression for neural networks.

Typical values: \(\lambda \in [10^{-5}, 10^{-2}]\). Too large → underfitting.

Dropout

During training, randomly set each neuron's output to zero with probability \(p\) (typically 0.5).

Each training step uses a different random mask.

  • Forces the network to learn redundant representations
  • No single neuron can be a point of failure
  • At test time: use all neurons, scale by \((1-p)\)
  • Ensemble view: averaging over \(2^H\) subnetworks

Batch Normalization

Normalize activations within each mini-batch, then scale and shift with learnable \(\gamma, \beta\):

\[\hat{z}_i = \frac{z_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}} \qquad y_i = \gamma\,\hat{z}_i + \beta\]
Why it helps:
  • Stabilizes each layer's input distribution
  • Allows higher learning rates
  • Mild regularizing effect
In practice:
  • Place between linear & activation: \(z \to \text{BN} \to g()\)
  • At test time: use running averages of \(\mu, \sigma\)
  • Nearly universal in CNNs

Key Takeaways

  • Shaping the trajectory: regularisation (weight decay, dropout, early stopping, batch norm) biases training toward features that generalise.
  • Weight decay = L2 penalty = Gaussian prior on weights. Encourages simpler functions.
  • Dropout randomly zeros neurons during training, forcing redundant representations. At test time, scale by \((1-p)\).
  • Batch normalisation stabilises layer inputs, enables higher learning rates, and acts as mild regularisation.
  • What networks learn: hidden layers progressively transform data into representations where the task becomes easy (linearly separable).
